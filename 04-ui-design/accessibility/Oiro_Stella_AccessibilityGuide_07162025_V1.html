Get a Room - Accessibility Framework

WCAG 2.1 AA Compliance Guide for Kenya Market

Designer: Stella Oiro | Date: July 16, 2025

Executive Summary

This accessibility framework ensures Get a Room's booking system is usable by all individuals, including those with disabilities. With 15% of Kenya's population having some form of disability, accessibility compliance is both a moral imperative and business opportunity.

Key Benefits

  • Market Expansion: Reach 7.5 million potential users in Kenya
  • Legal Compliance: Align with Kenya's disability rights legislation
  • Improved UX: Better usability for all users, not just those with disabilities
  • SEO Benefits: Better semantic structure improves search rankings

WCAG 2.1 AA Compliance Checklist

Perceivable

  • Color contrast ratio minimum 4.5:1 for normal text, 3:1 for large text
  • Alt text for all images, including M-Pesa logos and payment icons
  • Captions for any video content (demo videos, tutorials)
  • Text resizable up to 200% without horizontal scrolling
  • Images of text avoided where possible, use actual text instead
  • Color not used as the only means of conveying information
  • Audio controls for any background sounds or notifications

Operable

  • All functionality available via keyboard navigation
  • Tab order follows logical reading sequence
  • Touch targets minimum 44px x 44px for mobile devices
  • No content flashes more than 3 times per second
  • Users can pause, stop, or hide moving content
  • Skip links to main content and navigation
  • Session timeouts have warnings and extensions available
  • Focus indicators clearly visible for all interactive elements

Understandable

  • Language of page declared (en-KE or sw-KE)
  • Clear, simple language appropriate for literacy levels
  • Error messages specific and helpful
  • Instructions provided for complex interactions
  • Labels clearly describe form inputs
  • Consistent navigation throughout the application
  • Help text available for form fields requiring specific formats
  • Context changes only occur when expected by users

Robust

  • Valid HTML markup for proper screen reader interpretation
  • Semantic HTML elements used appropriately
  • ARIA labels and roles implemented correctly
  • Compatible with current and future assistive technologies
  • Content remains accessible when JavaScript is disabled

High Contrast Mode Specifications

Color Palette for High Contrast

Primary Colors

  • Background: #FFFFFF (White)
  • Text: #000000 (Black)
  • Links: #0066CC (Blue)
  • Visited Links: #6600CC (Purple)

Interactive Elements

  • Buttons: #000000 text on #FFFFFF background
  • Focus Outline: #FF0000 (Red), 3px solid
  • Error States: #CC0000 (Dark Red)
  • Success States: #006600 (Dark Green)

High Contrast CSS Implementation

@media (prefers-contrast: high) { :root { --primary-bg: #ffffff; --primary-text: #000000; --link-color: #0066cc; --button-bg: #ffffff; --button-text: #000000; --button-border: #000000; --focus-outline: #ff0000; --error-color: #cc0000; --success-color: #006600; } .btn-primary { background-color: var(--button-bg); color: var(--button-text); border: 3px solid var(--button-border); } .btn-primary:focus { outline: 3px solid var(--focus-outline); outline-offset: 2px; } .form-control { border: 2px solid var(--primary-text); background-color: var(--primary-bg); color: var(--primary-text); } .alert-error { background-color: var(--primary-bg); color: var(--error-color); border: 2px solid var(--error-color); } }

Reading Level Guidelines

  • Target 6th grade reading level for all interface text
  • Use active voice instead of passive voice
  • Keep sentences under 20 words when possible
  • Define technical terms on first use
  • Use bullet points instead of long paragraphs
  • Include visual cues to support text instructions

// Voice guidance for form errors const voiceMessages = { phoneNumberEmpty: "Phone number is required for M-Pesa payment", phoneNumberInvalid: "Please enter a valid Kenya phone number starting with 254", dateUnavailable: "Selected date is not available, please choose another date", paymentFailed: "Payment could not be processed, please try again or contact support", networkError: "Connection problem, please check your internet and try again" }; function announceError(errorType) { const message = voiceMessages[errorType]; // Use Web Speech API or screen reader announcement if ('speechSynthesis' in window) { const utterance = new SpeechSynthesisUtterance(message); utterance.lang = 'en-KE'; speechSynthesis.speak(utterance); } // Also update ARIA live region for screen readers document.getElementById('sr-announcements').textContent = message; }

Motor Disability Considerations

Touch Target Specifications

  • Minimum 44px x 44px touch targets for all interactive elements
  • 8px minimum spacing between touch targets
  • Alternative input methods for complex gestures
  • Large, easy-to-tap buttons for primary actions
  • Undo functionality for accidental activations
  • Confirmation dialogs for destructive actions

Mobile Accessibility Patterns

/* Mobile-first accessible touch targets */ .btn { min-height: 44px; min-width: 44px; margin: 4px; padding: 12px 16px; border-radius: 8px; font-size: 16px; line-height: 1.2; } .btn:focus { outline: 3px solid #0066cc; outline-offset: 2px; } /* Accessible form controls */ .form-control { min-height: 44px; padding: 12px; font-size: 16px; /* Prevents zoom on iOS */ border: 2px solid #666; border-radius: 4px; } .form-control:focus { border-color: #0066cc; box-shadow: 0 0 0 3px rgba(0, 102, 204, 0.3); }

Cognitive Load Reduction

Information Architecture Principles

Progressive Disclosure

  • Show only essential information per step
  • Use expandable sections for detailed info
  • Provide help text on demand
  • Hide advanced options initially

Memory Aids

  • Visual progress indicators
  • Breadcrumb navigation
  • Auto-save draft bookings
  • Recently viewed spaces

Error Prevention Strategies

  • Real-time form validation with helpful messages
  • Format examples for phone numbers and dates
  • Confirmation steps for irreversible actions
  • Clear distinction between primary and secondary actions
  • Contextual help available without leaving the page
